WorkGuru

Overview

WorkGuru is a MERN (MongoDB, Express, React, Node.js) stack project that provides a home service booking platform. Users can book services such as carpenters, plumbers, painters, movers, and packers. The project includes features for user authentication, OTP verification, and Google sign-in, along with a user-friendly interface and a comprehensive management system.

Prerequisites

Before you start, ensure you have the following installed:

  • Node.js (v14 or later)
  • MongoDB (for local development) or access to a MongoDB Atlas cluster
  • npm (Node Package Manager) or yarn (optional)

Setup Instructions

Clone the Repository

            git clone https://github.com/yourusername/WorkGuru.git
            cd WorkGuru

Install Dependencies

Install both server and client dependencies:

            # Server
            cd server
            npm install
            # Client
            cd ../client
            npm install

Running Locally

Server

            cd server
            npm start

The server will run on http://localhost:5000.

Client

            cd ../client
            npm start

The client will run on http://localhost:3000.

Deployment

Production Deployment

  1. Build the Client:
    2.                 cd client
                npm run build

    This will create a build directory with the production-ready files.

  2. Deploy the Server:
    • Push your server code to your Git repository.
    • Set up your hosting provider (e.g., Render) with the proper build and start commands.
    • Make sure to set the environment variables and database configurations.
  3. Deploy the Client:
    • You can serve the React build files from your server or deploy them separately to a service like Vercel or Netlify.
    • If deploying with Render, you can configure it to serve the build directory directly.

Environment Variables

Create a .env file in the server directory and add the following environment variables:

            MONGODB_URI=mongodb://localhost:27017/workguru
            JWT_SECRET=your_jwt_secret
            EMAILJS_USER_ID=your_emailjs_user_id
            EMAILJS_TEMPLATE_ID=your_emailjs_template_id
            EMAILJS_SERVICE_ID=your_emailjs_service_id

Ensure that you also set the corresponding environment variables in your deployment service.

Testing

Run tests for both the server and client:

            # Server
            cd server
            npm test
            # Client
            cd ../client
            npm test
